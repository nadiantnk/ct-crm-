## BUG-001: Скидка принимает значения больше 100% и отрицательные

**Приоритет:** 🔴 Критичный

### Шаги воспроизведения

1. Открыть расписание, кликнуть "+" "Записать пациента" на любом свободном слоте
2. Заполнить ФИО, телефон, выбрать услугу (например, «КТ двух челюстей — 3500₽»)
3. В поле «Скидка (%)» ввести 150
4. Посмотреть поле «Итого»

### Ожидаемый результат

Скидка ограничена диапазоном 0–100%. При вводе значения за пределами диапазона — показать ошибку или автоматически обрезать до допустимого значения.

### Фактический результат

Итоговая стоимость становится "отрицательной": -1750 ₽. Запись успешно сохраняется с минусовой ценой.

### Дополнительно

- При вводе **-50** цена становится выше базовой (5250 ₽ вместо 3500 ₽)
- Запись с некорректной ценой отображается в расписании и учитывается в отчётах

## BUG-002: Запись сохраняется с телефоном «+7» без номера

**Приоритет:** 🔴 Критичный

### Шаги воспроизведения

1. Открыть форму записи на любой свободный слот
2. Заполнить ФИО (например, «Тестов Тест»)
3. Поле телефона **не трогать** — там уже стоит «+7»
4. Выбрать любую услугу
5. Нажать "Сохранить"

### Ожидаемый результат

Валидация не пропускает запись без полного номера телефона. Минимум 11 цифр (формат +7 (XXX) XXX-XX-XX).

### Фактический результат

Запись успешно сохраняется. В расписании отображается пациент с телефоном "+7". Связаться с пациентом невозможно.

## BUG-003: Удаление клиники ломает существующие записи

**Приоритет:** 🟡 Средний

### Шаги воспроизведения

1. Записать пациента, указав клинику «Дентал Плюс»
2. Убедиться, что запись отображается в расписании с названием клиники
3. Перейти во вкладку «Клиники»
4. Удалить "Дентал Плюс"
5. Вернуться в расписание

### Ожидаемый результат

Варианты корректного поведения:
- Запретить удаление клиники, если есть связанные записи
- Показать предупреждение: "У этой клиники есть N записей. Удалить?"
- Сохранить название клиники в записи как текст

### Фактический результат

Клиника удаляется без предупреждения. В карточке пациента вместо названия клиники отображается «—». Данные о направляющей клинике потеряны безвозвратно.

## BUG-004: Удаление услуги ломает существующие записи

**Приоритет:** 🟡 Средний

### Шаги воспроизведения

1. Записать пациента на услугу "КТ ВНЧС" (3200 ₽)
2. Перейти во вкладку «Услуги»
3. Удалить "КТ ВНЧС"
4. Вернуться в расписание

### Ожидаемый результат

Аналогично BUG-003 — запретить удаление или предупредить.

### Фактический результат

Услуга удаляется. В карточке пациента вместо названия услуги — «—». Цена в карточке может отображаться некорректно. При открытии записи на редактирование — поле услуги пустое.

## BUG-007: Поле скидки принимает нечисловые значения

**Приоритет:** 🟢 Мелочь

### Шаги воспроизведения

1. Открыть форму записи
2. В поле "Скидка (%)" попробовать ввести: буквы, спецсимволы, дробное число (12.5)
3. Посмотреть на итоговую стоимость

### Ожидаемый результат

Поле принимает только целые числа от 0 до 100.

### Фактический результат

Поле блокирует буквы, но принимает значения +, - и .

## BUG-008:Поле "Дата рождения принимает любую дату без валидации

**Приоритет:** 🟡 Средний

### Шаги воспроизведения

1. Открыть форму записи
2. В поле "Дата рождения" попробовать ввести 08.11.1590
